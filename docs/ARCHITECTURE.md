# 아키텍처

## 개요
무역풍은 **데이터 파이프라인 → AI 엔진 → 표현(웹/앱/B2G)** 의 3계층으로 구성된다.
핵심 설계 원칙은 두 가지다.

1. **데모는 항상 동작한다** — 외부 인증키 없이도 재현 가능한 스냅샷과 미리 구운 JSON으로 정적 PWA가 완결된다(오프라인).
2. **AI는 근거와 함께 말한다** — 모든 예측·상담 결과에 실제 수출 수치를 동반해 환각을 차단한다.

```
┌─────────────────────────────────────────────────────────────────┐
│ 1) 데이터 계층                                                     │
│   관세청 OpenAPI(nitemtrade) ──customs_client──▶ snapshot.json     │
│   (실서비스: 인증키 동기화 / 데모: 공표치 앵커 스냅샷, 시드 고정)      │
├─────────────────────────────────────────────────────────────────┤
│ 2) AI 엔진 계층 (numpy, 100% 국산·자체)                            │
│   forecasting.py  승법 계절분해 + 감쇠 로그선형 추세 외삽            │
│   radar.py        모멘텀·가속도·변동성 → 기회/리스크 스코어·조기경보 │
│   advisor.py      의도추출 + 근거검색 NLG (+ 국산 LLM 어댑터)        │
├─────────────────────────────────────────────────────────────────┤
│ 3) 표현 계층                                                       │
│   build_app_data.py ─▶ app/data/*.json ─▶ 정적 PWA(웹앱·대시보드)   │
│   server/main.py(FastAPI) ─▶ /api/* 실시간 + 정적 동시 서빙          │
│   packaging/android: Compose 네이티브 + TWA                        │
└─────────────────────────────────────────────────────────────────┘
```

## AI 엔진 상세

### 수요예측 (forecasting.py)
월별 수출 시계열 `y[t]`에 대해:
1. **승법 계절지수** 추정 — 주기 12, 2주기 이상 평균비로 초기화 후 평균 1.0 정규화.
2. **탈계절화** `d[t] = y[t] / seasonal[t mod 12]`.
3. 최근 `W`(기본 18)개월 `log(d)`에 **로그선형 회귀** → 현재 국면의 월 성장률 `g`와 추세선상의 끝점(level)을 추정(추세 시차 제거).
4. **감쇠 외삽** `level · Π(1 + g·φ^h) · seasonal` — 감쇠계수 φ로 성장률을 점차 줄여 폭주·평균회귀를 모두 방지.
5. 인-샘플 잔차로 95% 예측구간(`±1.96·σ·√h`)과 적합도(MAPE) 산출.

설명 가능(explainable)하고 경량이라 온디바이스·국내 인프라에서 그대로 구동된다.

### 조기경보 (radar.py)
국가×품목별로 단기성장(3개월)·YoY·로그추세 기울기·가속도(기울기의 변화)·변동계수를 추출하고,
예측 엔진의 향후 전망을 결합해 **기회점수(0~100)** 와 **리스크점수**를 산출한다.
상태는 `급등/상승/안정/둔화/변동`으로 분류하며, 규모 있는 시장의 둔화는 리스크 경보로 띄운다.

### AI 무역참모 (advisor.py)
한국어 질문에서 품목·국가·의도(추천/예측/리스크/개요)를 추출 → 예측·레이더 결과를 **근거 팩**으로 묶어 답변.
데모는 결정적 NLG로 오프라인 동작하고, 운영은 동일 근거 팩을 **국산 LLM**(Solar·HyperCLOVA X) 어댑터에 넘겨 문체만 다듬는다(근거는 고정 → 환각 차단).

## 기술 스택
- 백엔드/AI: Python · FastAPI · numpy
- 프런트: 정적 PWA(Vanilla JS · 인라인 SVG 차트, 무프레임워크 → 경량·오프라인)
- 앱: PWA(TWA) + Jetpack Compose(Kotlin)
- 배포: GitHub Pages(정적) 또는 Render/Docker(FastAPI)

## 확장 로드맵
- 실데이터 일배치 동기화(Airflow/cron) + HS 10단위 세분화
- HS코드 자동분류(상품명/사진), FTA 원산지·관세율 융합으로 landed-cost 제공
- 국산 LLM 정식 연동, 알림(둔화 경보) 푸시
